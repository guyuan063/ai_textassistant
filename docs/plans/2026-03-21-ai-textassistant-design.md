# AI 智能文本助手设计说明

## 目标

构建一个入门级、易实现的 AI 文本助手，覆盖润色、翻译、摘要、创意文案 4 类高频文本处理场景，并提供一键复制、模板保存与历史记录能力。

## 方案选择

采用单页工作台方案，而不是拆分多页面路由：

- 学习成本更低，适合新手快速理解
- 标签页切换足够承载 4 类功能
- 右侧统一放模型配置、模板和历史，交互更集中

## 技术结构

- `Vue 3 + Vite`：项目骨架与组件开发
- `Element Plus`：表单、标签页、按钮、卡片等基础 UI
- `Pinia`：统一管理输入文本、当前任务、模型配置、结果、历史、模板
- `Axios`：封装 OpenAI 与百度千帆 / 文心 API 请求
- `LocalStorage`：持久化模板、历史与最近一次表单配置

## 模块划分

- `WorkbenchPanel`：任务标签页、输入区、任务参数
- `ResultPanel`：结果展示、复制、错误提示
- `SidebarPanel`：模型设置、模板库、历史记录
- `assistant store`：状态中心
- `ai service`：Prompt 生成与 API 请求适配

## 风险与边界

- 前端直连 API 仅适合演示，不适合生产
- 百度接口可能因账号权限差异需要手动调整模型名或地址
- 浏览器环境下跨域与密钥泄露需要特别注意
